## 도서명: 그림과 실습으로 배우는 도커&쿠버네티스

### SSH
- 원격으로 컴퓨터에 접속
- 윈도우로 따지면 원격 데스크톱(그러나 GUI가 아니다)
- SSH 소프트웨어 예: PuTTy

### ECS, EKS
- AWS의 커메이너 오케스트레이션 서비스 (여러 컨테이너를 자동으로 관리)
- 컨테이너 배포, 장애 복구, 로드 밸런싱, 자동 확장 등
- ECS: AWS의 자체 컨테이너 오케스트레이션 서비스/ 간단/ 무료
- EKS: AWS의 관리형 Kubernetes 서비스/ 큰 커뮤니티/ 복잡/ 비용 높음
- 프로덕션용 (개발/테스트용은 Docker Compose)
- 순서: Docker 기초 -> Docker Compose -> Kubernetes 기초 -> EKS

### 도커 사용의 3대 핵심 이유
1. 환경 일관성
2. 격리
3. 이식성 (개발자는 Docker 이미지를 빌드한다 -> 테스트팀은 같은 이미지를 실행한다 -> 운영팀도 같은 이미지를 실행한다)

### Tomcat Container VS Docke Container
- Tomcat Container(애플리케이션 컨테이너)= Java 웹 애플리케이션을 실행하는 소프트웨어
- Docker Container(OS-레벨 컨테이너)= 운영체제를 가상화하는 격리 환경
- 격리 범위가 Docker는 운영체제 전체, Tomcat은 Java 애플리케이션만
- (예) Docker 이미지에 Tomcat+WAR 포함하여 환경 일관성 보장
