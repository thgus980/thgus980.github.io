## RAG 성능 개선

### 참고 문서 표시: S3 Pre-signed URL
- 장점: 보안 우수, 추가 인프라 불필요, 문서가 S3에 비공개 상태로 유지됨
- 단점: URL에 유효 기간이 있음
- 원래 사내 문서는 비공개 S3에 저장되어 있어 사용자는 이 파일들에 직접 접근할 수 없음(S3 버킷이 비공개 상태이므로 AWS 자격 증명이 필요, 사용자의 브라우저는 AWS 인증을 할 수 없음)
- 해결책은 Pre-signed URL -> AWS 자격 증명을 노출하지 않고도, 비공개 S3 객체에 접근할 수 있는 권한을 부여하는 임시 보안 URL
- 주요 특징: 일시적, 보안, 자격 증명 비노출, 일회성 생성
