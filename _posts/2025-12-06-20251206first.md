## 도서명: 그림과 작동 원리로 쉽게 이해하는 AWS 구조와 서비스

### 키워드 중심 정리
- 클러스팅: 여러 대의 컴퓨터가 연결돼 하나의 시스템처럼 동작하는 컴퓨터들의 집합
- 서버리스 방식: 서비스가 이용될 때만 서버를 가동하는 방식
- 프로비저닝 방식: 서버가 계속 실행 중이므로 사용자는 언제든지 사용할 수 있다. 실제 이용하지 않더라도 서버가 동작하고 있다면 그 시간만큼 요금이 발생한다.
- PasS, IaaS: PaaS에는 Lambda(서버+미들웨어), IaaS에는 EC2(서버)
- EC2: 아마존의 가상 서버 서비스, 가상 서버를 인스턴스 단위로 관리
- EC2의 Auto Scaling: 서버를 추가하는 것을 스케일 아웃, 서버를 제거하는 것을 스케일 인
- 가상 서버 이미지 VS 컨테이너 이미지: 가상 서버 이미지는 OS, 미들웨어, 응용 프로그램을 포함하고 있음, 컨테이너 이미지는 응용 프로그램만 포함하고 있음
- S3: 아마존의 데이터 저장소, 객체 스토리지(객체 키로 고유하게 식별해 데이터의 입출력과 관리 수행) 서비스, 인터넷 연결되어야 함 (API 호출 방식으로 사용)
- EBS: EC2와 함께 사용하는 스토리지 서비스, EC2의 HDD/SSD와 같은 역할을 함
- VPC: AWS에서 생설할수 있는 프라이빗 가상 네트워크 공간, VPC안에 EC2가 있는 것

### 컨테이너 이미지는 여러 시스템 환경에 쉽게 배포 가능
1. PC 및 서버에는 컨테이너 런타임을 설치해 둔다
2. 로컬 환경에서 응용 프로그램을 개발
3. 컨테이너 이미지 생성
4. 이미지를 레지스트리에 업로드(푸시)
5. 레지스트리에서 이미지를 설치(풀)
6. (상용 환경과 개발 환경 등이 있는 시스템 환경) 컨테이너 런타임이 있으면 같은 컨테이너 이미지를 사용해 로컬에서 개발한 응용 프로그램을 동일하게 실행 가능 (복수의 환경에도 쉽게 배포 가능, 재현성이 높으므로 환경에 의존하지 않음, OS 및 하드웨어를 동일하게 맞추지 않아도 된다)
