## AI

### Amazon Bedrock
- AWS에서 제공하는 생성형 AI 모델을 API 형태로 쉽게 가져다 쓸 수 있는 완전 관리형 서비스
- 다양한 파운데이션 모델 제공 (Claude, Llama ...)
- 인프라 관리 불필요: 직접 GPU 서버를 띄우거나 모델을 설치/튜닝할 필요 없음. 그냥 API만 호출.
- GUI에서 만든 것(AWS 콘솔 웹)을 코드로 호출만한다
- [AWS 웹 콘솔]: AWS 콘솔(웹)의 GUI 상에서 Knowledge Base(Bedrock 콘솔의 메뉴) 생성 -> ID 발급
- [개발자]: Python 코드 작성 -> bot3 라이브러리 함수에 ID 입력
- [사내 시스템]: 직원이 질문하면 -> Python 코드가 실행되고 -> ID를 통해 AWS에 있는 RAG Data를 검색해서 답변 가져옴
```python
  import boto3

  # 베드락 클라이언트 연결
  bedrock_agent_runtime = boto3.client('bedrock-agent-runtime', region_name='us-east-1')
  
  def ask_hr_bot(user_question):
      response = bedrock_agent_runtime.retrieve_and_generate(
          input={
              'text': user_question  # 예: "내 남은 연차 며칠이야?"
          },
          retrieveAndGenerateConfiguration={
              'type': 'KNOWLEDGE_BASE',
              'knowledgeBaseConfiguration': {
                  # ▼ 여기가 바로 GUI랑 연결되는 핵심 연결고리! ▼
                  'knowledgeBaseId': 'ABC-12345-XYZ', 
                  
                  # 모델도 GUI에서 정한 걸 쓰겠다고 명시
                  'modelArn': 'arn:aws:bedrock:us-east-1::foundation-model/anthropic.claude-3-sonnet' 
              }
          }
      )
    
    return response['output']['text']
```

### Docker와 SSH
- 비유적으로 표현하자면, 서버는 아파트 건물 전체, SSH는 아파트 건물 현관문 열쇠, 도커 컨테이너는 아파트 안의 각 호실 (101호, 102호 ...)
- 서버에 들어갈 때 SSH를 쓴다
- 컨테이너 내부에 들어갈 때: `docker exec`를 쓴다
