## AWS 교육

### Provisioning
- IT 리소스를 준비하고 제공하는 것, 사용 가능한 상태로 만드는 것
- 리소스 준비 과정. (생성: 리소스 만들기 -> 설정: 필요한 옵션 지정 -> 연결: 다른 리소스와 연결 -> 준비완료: 사용 가능 상태)

### CIDR
- 사이더
- IP 주소 범위를 표시하는 방법
- 형식: IP주소/숫자
- 192.168.1.0/24 -> 192.168.1.0~192.168.1.255 (앞 24비트 고정) -> 256개 IP, 주로 용도는 작은 사무실
- VPC 하나에 여러 서브넷 분할 시 사용하는 표시 방법 (서브넷 분할 시, 하나의 큰 IP 범위를 작은 조각으로 나눈다)

### Internet Gateway
- IGW
- VPC와 인터넷 사이 존재 (양방향) -> VPC와 인터넷을 연결하는 출입구

### NAT Gateway
- Private Subnet에서 인터넷으로 (단방향) -> 나가기만 가능
- NAT Gateway는 **나가기**만 가능하다 -> 외부에서 들어오는 것은 절대 불가
- apt update, pip install -> 이것들과 같이 Priavte에서 인터넷으로 나가기 가능
- 외부에서 SSH나 HTTP 접속 -> 인터넷에서 Private로 들어오기 불가능
- 그렇다면 **외부에서 Private Subnet 들어가는 방법**에는 무엇이 있을까?
    - Bastion Host(점프 서버)
    - VPN
    - AWS System Manager (SSM)
    - ALB
    - 정리: Public Subnet의 Bastion Host에서 Private Subnet로 접근할 때 SSH, Piravte Subnet에서 Public Subnet으로 나갈 때는 Public Subnet의 NAT Gateway를 거친다

### VPC Endpoint
- VPC와 AWS 서비스 사이 (인터넷 거치지 않음)
- 예를 들어 S3 접근을 더 쉽고 빠르게 할 수 있음 (인터넷을 거치지 않고 VPC Endpoint로 접근하기 때문에)

### AMI
- Amazon Machine Image(AMI)
- 구성요소: 인스턴스 볼륨 템플릿, 시작 권한, 블록 디바이스 매핑
- 이점: 반복 가능, 재사용 가능, 복구 가능
