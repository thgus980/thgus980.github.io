## AWS Certified Solutions Architect – Associate (SAA)

### Application Load Balancer (ALB)
- Q. 여러 가용 영역(Multi-AZ)에 걸쳐 실행되는 EC2 인스턴스에서 웹 애플리케이션을 운영하고 있습니다. 특정 서버에만 트래픽이 몰리지 않도록 모든 인스턴스에 부하를 균등하게 분산하고, 특정 서버가 고장 났을 때 해당 서버로 트래픽이 가지 않도록 차단하고 싶습니다. 어떤 서비스를 사용해야 합니까?
    1. Amazon Route 53
    2. Amazon CloudFront
    3. Application Load Balancer (ALB)
    4. AWS Auto Scaling
- A. 3번 Application Load Balancer (ALB)
    - ALB(Application Load Balancer): 여러 대의 **EC2 인스턴스** 앞에서 '교통정리'를 해주는 서비스. 사용자의 요청(HTTP/HTTPS)을 받아서 뒤에 있는 서버들에게 골고루 나눠줌.
    - 상태 확인(Health Check): ALB는 주기적으로 서버가 살아있는지 확인. 만약 서버 하나가 응답하지 않으면(장애 발생), 즉시 그 서버를 목록에서 제외하고 정상적인 서버로만 트래픽을 보냄.
    - Amazon Route 53: 도메인 이름을 IP로 바꿔주는 전화번호부 같은 서비스
    - CloudFront: 파일 전송 속도를 빠르게 해주는 캐시 서비스
    - Auto Scaling: 서버의 개수를 조절할 뿐, 트래픽을 직접 나눠주지는 않음.

### ALB와 Auto Scaling
- Auto Scaling이 트래픽에 맞춰 서버를 5대로 늘리면, ALB가 그 5대에게 트래픽을 골고루 보내주는 식
- ALB는 교통 경찰, Auto Scaling은 인력 관리자처럼 항상 세트로 움직임
- 살아있는 서버로만 길을 안내하는 똑똑한 ALB!
