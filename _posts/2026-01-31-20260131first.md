## 로그인 통합을 위한 Python Backend Framework

### 종류
- Django: 웹 개발에 필요한 거의 모든 기능(로그인, 관리자 페이지, ORM, 보안 등)이 이미 내장되어 있어 설정할 것이 적음/ 프레임 워크가 무겁고 배우는 데 시간이 걸림 -> 커뮤니티 사이트, 이커머스, 관리자 기능이 중요한 서비스에 적합
- FastAPI: 비동기 처리에 최적화, 자동 문서화 -> MSA, 데이터 과학 모델 서빙 등에 적합
- Flask: 최소한의 기능 제공 -> 소규모 토이 프로젝트에 적합

### 로그인 통합을 위해서 Django vs Fast API?
- Django: 내장 ORM, 자동 관리자 페이지, 인증 및 권한 시스템, 보안 기능 등 거의 모든 도구가 이미 규격화 되어 있어 편리함
- FastAPI: 개발자가 직접 선택해서 조립해야 함(예: 로그인 기능을 구현할 때 `fastapi.security`를 사용, 관리자 페이지는 `FastAPI-amis-Admin`을 사용, ORM은 SQLAlchemy를 사용 등)

### SSO 구현을 위한 방법 1
- Django가 Proxy 역할을 수행 -> Django를 인증 서버로 활용
  1. 사용자가 Django 웹페이지에 접속하여 사내 SSO로 로그인
  2. Django는 사용자의 세션을 확인하고 **웹 화면(React/Vue 또는 Django Template)**을 띄움
  3. 사용자가 채팅을 입력하면, 브라우저는 Django를 거치거나 혹은 Django가 발급해 준 토큰을 가지고 FastAPI 서버로 직접 요청
- Django는 오직 로그인과 권한 확인만 담당하게 하는 방법. Django에서 로그인 성공 시 JWT 토큰을 발급하고, 이 토큰이 있어야만 FastAPI 챗봇 API를 호출할 수 있게 설계하는 방식

### SSO 구현을 위한 방법 2
- Reverse Proxy 활용
- Nginx 같은 서버를 앞단에 두고, /auth는 Django로, /api/chat은 FastAPI로 보내주도록 설정하면 사용자 입장에서는 하나의 웹사이트처럼 보임
