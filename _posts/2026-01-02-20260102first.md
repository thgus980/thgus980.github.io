## AWS 잘하는 개발자 되기

### 컨테이너 서비스, 아마존 ECS
- Elastic Container Service는 AWS에서 도커 컨테이너를 배포하고 운영, 관리하는 완전 관리형 컨테이너 서비스
- 도커는 환경 불일치를 해결하는 오픈 소스 프로젝트
- 먼저 사용자는 로컬 환경 혹은 개발을 진행하고자 하는 서버에 도커를 설치 -> 사용자가 구성하고자 하는 환경을 바탕으로 도커 파일을 작성 -> 작성한 도커파일을 다른 서버로 업로드하고 이 서버에도 동일하게 도커를 설치한 다음, 업로드된 도커파일을 빌드해 같은 개발 환경을 구성
- 이렇게 구성한 가상의 개발환경을 컨테이너라고 한다
- 직접 사용자가 도커파일을 만드는 방법도 존재하지만 도커허브에 접속해 다른 사람들이 만들어둔 도커파일을 다운로드해 개발 환경을 구성할 수도 있음

### 아마존 API 게이트웨이
- 애플리케이션은 API 게이트웨아와 통신하며, API 게이트웨이는 API를 제공하는 애플리케이션 혹은 서비스와 통신하므로 다양한 애플리케이션에 대한 특정 요청을 조합하거나, API에서 발생하는 모든 것을 API 게이트웨이에서 모니터링해 관리할 수 있음
- 구성요소: REST API, REST API 프라이빗, HTTP API, 웹소켓 API

### HTTPS와 로그 설정
- HTTPS를 사용하려면 라우트53에 도메인을 등록하고, AWS ACM을 이용해 인증서를 발급받는다. 이렇게 발급받은 인증서를 로드 밸런서에 등록함으로써 HTTPS 포트를 사용


## Amazon Bedrock으로 시작하는 실전 생성형 AI 개발

### bedrock이란
- Bedrock 주요 기능: 다양한 기반 모델 액세스, 모델 테스트 환경 제공, 간편한 모델 미세조정, AI 에이전트 구축, RAG 지원, 서버리스 추론, 보안 및 개인정보 보호, 책임 있는 AI, 서버리스 확장성 및 비용 효율성

### bedrock agent의 작동순서
1. 사용자의 입력이 bedrock agent로 전달
2. bedrock agent는 사용자 입력을 다음 요소들과 결합 (미리 정의된 프롬프트, 이전 대화 기록, Lambda 함수에 대한 정보, 지식 기반에 대한 정보, 수행해야 할 작업에 대한 지시)
3. 결합된 정보가 LLM에 입력으로 제공
4. LLM은 제공된 정보를 바탕으로 사고 과정을 수행 (사용자 입력을 분석하여 적절한 작업 선택, 필요에 따라 API 호출 또는 RAG 수행)
5. 원하는 결과가 나올 때까지 이 과정 반복
