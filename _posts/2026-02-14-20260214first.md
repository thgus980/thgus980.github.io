## RPA 고도화 방안

### 툴 의존도를 낮추는 '커스텀 스크립트' 삽입 (Python/C#)
- 내부적으로 Python이나 .NET 코드를 실행할 수 있는 블록을 사용하기. 복잡한 데이터 가공이나 로직은 툴의 액션(Action)이 아닌 직접 짠 코드로 처리.
- 구현 방법: 엑셀의 수만 행을 루프(Loop) 돌리는 대신, Python의 Pandas 라이브러리를 사용해 메모리상에서 순식간에 처리한 뒤 결과만 반환하게 만들기.
- 기대 효과: 실행 속도가 압도적으로 빨라지며, 툴 업데이트에 따른 로직 깨짐 현상(Side Effect)을 최소화.

### 'API-First' 자동화 (GUI 클리킹 탈피)
- 화면을 일일이 클릭하는 방식은 UI 요소가 조금만 바뀌어도 에러가 발생. 가능하다면 웹 화면이 아닌 **내부 API(REST API)**를 직접 호출하는 방식으로 고도화.
- 구현 방법: RPA 툴 내에서 HTTP Request 기능을 사용하여 백엔드 서버와 직접 데이터를 주고받기.
- 기대 효과: 속도는 물론이고 안정성이 비약적으로 상승. "화면이 안 보여서 에러 났다" -> **"API 응답 값 예외 처리 완료"**

### 중앙 집중식 '로그 분석 및 DB화' (PostgreSQL 활용)
- 모든 수행 기록을 PostgreSQL과 연결.
- 구현 방법: RPA가 시작/종료될 때, 혹은 에러가 발생할 때마다 DB에 쿼리를 날려 로그를 쌓기.
- 기대 효과: Dashboard 구축-> Grafana나 별도의 웹 대시보드를 만들어 "이번 달 총 절감 시간", "성공률" 등을 시각화.
- 예측 유지보수: 특정 시간대에 에러가 잦다면 이를 데이터로 분석해 인프라 문제를 먼저 찾아낼 수 있음.

### 'Cognitive RPA'로의 진화 (AI API 연동)
- 단순 규칙(If-Then)을 넘어 **판단**이 필요한 영역에 AI를 이식
- 구현 방법: 비정형 데이터 처리 -> PDF나 이미지에서 텍스트를 추출할 때 단순 OCR이 아닌 **LLM(Claude/GPT)** 을 연동해 맥락을 파악하여 데이터를 분류.
- 감성 분석 기반 자동 대응: 고객 문의 메일을 AI가 분석해 화가 난 고객의 메일만 골라 우선적으로 담당자에게 알림을 주는 식.
- 기대 효과: "단순 툴 사용"이라는 인식을 깨고 **"AI 결합 자동화 시스템"** 이라는 독보적인 브랜딩이 가능해짐.
