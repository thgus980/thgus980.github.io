## 클로드 코드 가이드: Skills

### 스킬이란?
- CLAUDE.md가 프로젝트 전체에 적용되는 규칙이라면, 스킬은 특정 작업을 위한 플레이북
- Claude의 기능을 확장하는 재사용 가능한 지시사항
- SKILL.md 파일로 작성
- /skill-name으로 직접 호출 가능
- (예) 이슈를 수정하는 방법을 스킬로 정의해 두면, /fix-issue123만 입력해서 이슈 확인부터 커밋까지의 전체 워크플로우를 실행할 수 있음
- 예를 들어 빌드 후 특정 서버 업로드, 테스크 자동화 등...

### 번들 스킬(별도 설정 없이 바로 쓸 수 있는 스킬)
1. /simplify: 최근 변경된 파일의 코드 재사용성, 품질, 효율성을 검토한 후 수정.
2. /batch <instruction>: 코드베이스 전반에 걸친 대규모 변경을 병렬로 수행. (/batch src/를 Solid에서 React로 마이그레이션)
3. /debug [description]: 현재 Claude Code 세션의 디버그 로그를 읽어 문제를 해결.

### 스킬 파일 예시
  ```markdown
  ---
  name: explain-code
  description: 코드를 시각적 다이어그램과 비유로 설명합니다. "어떻게 동작해?"라고 물을 때 사용합니다.
  ---
  
  코드를 설명할 때 항상 다음을 포함하세요:
  
  1. **비유로 시작**: 코드를 일상생활의 무언가와 비교
  2. **다이어그램 그리기**: ASCII 아트로 흐름, 구조, 관계 표시
  3. **코드 워크스루**: 단계별로 무슨 일이 일어나는지 설명
  4. **주의사항**: 흔한 실수나 오해 강조
  ```

### 정리
- claude.md는 claude에게 "어떤 존재가 되어야 하는지"를 가르치고, skills.md는 "무엇을 할 수 있는지"를 가르치는 것
- claude.md는 지침서, skills.md는 기술교본
- claude.md는 작동 원리면에서 대화 내내 배경 지식으로 깔려 있는 것이고, skills.md는 특정 상황이나 요청이 있을 때 호출됨
- claude.md는 실행 트리거면에서 모든 답변 생성 시 자동 적용이고, skills.md는 관련 키워드 언급 또는 명령어 입력 시
