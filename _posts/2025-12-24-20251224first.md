## AWS 잘하는 개발자 되기

### AWS를 쓰려면 알아두면 유용한 클라우드 지식
- 온프레미스: 조직이 자체적으로 IT 인프라를 소유하여 관리 및 운영하는 것(회사 안에 데이터 센터가 있고 데이터 센터 안에는 서버, 스토리지 및 네트워크 장비가 있음)
- 클라우드: 가상공간에 서버를 구축(온프레미스의 불편한 점 해소)
- AWS 글로벌 인프라 구성은 리전, 가용 영역, 에지 로케이션으로 구성
- 가용영역: AWS 리전 내에 서버, 스토리지, 네트워크 장비를 저장하는 하나 이상의 물리적인 개별 데이터센터로 구성됨
- 에지 로케이션:오리진 서버의 콘텐츠를 서로 다른 지역에서 빠르게 접근할 수 있도록 만드는 캐시 서버의 모음(AWS는 에지 로케이션을 이용하여 아마존 클라우드프론트라는 CDN 서비스를 제공)

### IP 주소 체계- 네트워크 규모로 나눈 IP 클래스
- IP주소는 네트워크 규모에 따라 A~E 클래스로 나누어져 있음. (A 클래스는 대규모 네트워크, B 클래스는 중형, C 클래스는 소규모)
- 예를 들어, 회사에서 C 클래스의 192.168.1.0 네트워크 주소를 할당했다면 네트워크 주소(192.168.1.0)과 브로드캐스트 주소(192.168.1.255)를 제외하고 사용 가능

### 아마존 VPC 구성 요소 살펴보기
- 서브넷: 서브넷은 아마존 VPC 내에서 네트워크를 더 작은 단위로 나눈 부분 네트워크
- 인터넷 게이트웨이: 인터넷 게이트웨이는 VPC와 인터넷 간의 통신을 가능하게 하는 네트워크 게이트웨이
- NAT 게이트웨이: 프라이빗 서브넷 내의 리소스가 VPC 외부 서비스에는 연결할 수 있지만 외부 서비스에서 이 리소스와는 연결을 할 수 없도록 함 (퍼블릭 서브넷에 존재)
- 라우팅 테이블: VPC 안에서 발생한 네트워크 요청을 어디로 전송해야 하는지 알려주는 역할
- VPC 엔드포인트: VPC 내 리소스들이 S3, 다이나모DB와 같이 다른 AWS나 외부 리소스에 접근할 때 내부 네트워크를 통해 안전하게 접근할 수 있도록 도와줌
