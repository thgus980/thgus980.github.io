## AWS Certified Solutions Architect – Associate (SAA)

### Amazon EC2 Auto Scaling
- Q. 서버가 특정 시간대에만 트래픽이 몰립니다. 비용을 아끼기 위해 사용자가 없을 때는 서버 수를 줄이고, 사용자가 많아지면 자동으로 서버 수를 늘려주는 서비스는 무엇일까요?
  1.  AWS CloudTrail
  2.  Amazon EC2 Auto Scaling
  3.  AWS Config
  4.  Amazon CloudFront
- A. 정답 2번
  1. AWS CloudTrail (감사/기록): "누가, 언제, 어떤 API를 호출했나?"를 기록하는 로그 시스템. 서버 수를 조절하는 기능은 없음. (예: "어제 누가 EC2 인스턴스를 삭제했지?"를 찾을 때 사용)
  2. AWS Config (설정 관리): 리소스의 설정 상태를 기록하고 가이드라인을 준수하는지 체크하는 도구. (예: "모든 S3 버킷이 암호화되어 있는가?"를 확인)
  3. Amazon EC2 Auto Scaling: 자동으로 규모를 조절하는 서비스
  4. Amazon CloudFront (배포/캐싱): 전 세계 사용자에게 콘텐츠를 빠르게 전달하기 위한 CDN 서비스. 정적 파일(이미지 등)을 캐싱해서 응답 속도를 높여주지만, 서버의 대수를 직접 조절하지는 않음.
 
### 참고
- Auto Scaling은 말 그대로 자동으로 규모를 조절하는 서비스. 트래픽이 많을 때, 서버 EC2가 과부하로 죽지 않도록 자동으로 인스턴스 개수를 늘림(Scale-out). 트래픽이 적을 때, 놀고 있는 서버를 자동으로 종료시켜서 불필요한 비용이 나가지 않도록 함(Sacle-in).
- Auto Scaling이 어떤 기준으로 작동하는가?
  1. CPU 사용률: "CPU가 70%를 넘으면 서버 한 대 더 추가해줘!"
  2. 대상 추적 정책: "평균 CPU 사용률을 항상 50%로 유지해줘!"
