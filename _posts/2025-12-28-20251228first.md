## 챗봇 아키텍처

### AWS 서버리스와 Frontend
- Frontend를 서버리스로 배포하기 (React+S3+CloudFront)
    1. 로컬PC에서 React 파일 bulid
    2. 생성된 bulid 폴더 안의 파일들을 S3 버킷에 업로드
    3. CloudFront를 생성하고 S3와 연결
    4. 사용자는 CloudFront가 제공하는 주소로 접속하면 챗봇 화면이 뜸
- Frontend 서버가 따로 필요없는 이유? 자바스크립트 코드가 실행되는 장소는 **사용자의 브라우저** (브라우저로부터 접속 요청이 오면 CloudFront가 S3에 있는 React 파일을 브라우저로 던져줌 -> 브라우저가 방금 받은 JS 파일을 읽음)

 ### AWS App Runner와 WebSocket
 - App Runner+WebSocket 방식으로 갈 경우에 서버는 계속 클라이언트 브라우저로부터 오는 응답을 대기해야하기 때문에 진정한 서버리스 구현은 불가능 (WebSocket은 지속적인 양방향 연결)
 - 대안: API Gateway WebSocket+Lambda
     - AWS의 완전 관리형 서비스가 연결을 관리 (서버인 Lambda는 대기하지 않음, 메시지 올 때만 Lambda를 깨움)
     - 진정한 서비리스 구현 가능 (AWS App Runner+WebSocket 조합은 AWS가 인프라를 관리해준다는 면에서 관리형 서버정도)
     - API Gateway가 대신 WebSocket 연결 유지, 메시지 수신 대기
