## AWS 교육

### 키워드 중심으로 정리
- EC2 Spot Instances  
학습 서버는 Spot Instance, 추론 서버는 On-Demand

- EC2(GPU) VS Lambda(서버리스)  
가벼운 작업은 서버리스로 전환

- Auto Scaling  
트랙픽 양에 따른 EC2 가동 수 조정

- S3  
대표적 클라우드형 저장소.  
정형, 비정형 상관 없이 Object 스토리지 서비스.  
온프레미스 데이터를 클라우드로 이동하려고 할 때 많이 사용함. (클라우드 환경 이전)  
클라우드 환경 이전 시, Date Lake로 사용됨.  

- FSx  
파일 공유 시스템, 초고속 읽기 가능하여 AI 학습에 효율적, 학습데이터 I/O 효율적

- Aurora Pgvector  
Pgvector를 사용할 수 있게 하는 AWS 서비스 (Vector DB)

- 계층적 DataLake 구조  
Raw Zone, Cleaned Zone, Curated Zone, Archive Zone -> 각 Zone마다 최적화 할 수 있는 S3 스토리지를 정한다

- AI는 벡터로 생각한다  

- VPC, CloudeFront, IAM, KMS  
VPC: 고객들별로 고객만의 가상의 private cloud 환경을 제공해주겠다  
KMS: 암호화 하는 데 필요한 Key를 관리하는 시스템 (자동으로 키가 만들어지고, S3 버킷은 자동으로 암호화가 되고, EBS 암호화, 키의 노출을 최소화 하기 위해 매년 자동적으로 키를 다시 만들고 배포)  
CloudeFront: AWS의 CDN 서비스 (콘텐츠 배포 네트워크, 전 세계 캐시 서버 네트워크 -> 글로벌 사용자들이 가까운 서버에서 빠르게 받음)  

- Amazon CloudFront(CDN)  
글로벌 캐시 서비스, 글로벌하게 서비스를 확장할 수 있음  
예) 한국 S3에 영상이 올라와있는데 미국 사용자가 해당 영상을 보기 위해서는 US Edge Location(한국 S3데이터 캐싱한 곳)으로부터 가져온다 (굳이 미국 사용자가 한국 리전까지 오지 않아도 된다)  
-> 사용자에게 가장 가까이 있는 캐시 저장소에서 데이터를 받아갈 수 있음 -> 속도 매우 유리  

- AI 모델이 유출되면 회사는 끝  
공인 IP를 최대한 사용하지 말자  
DOS 공격 차단 서비스 -> WAF  
Access key 깃허브에 노출 절대 금지 (AWS에서 내리라고 전화 옴)  
S3의 특벙 버킷을 public하게 설정하는 경우 위험하다  
EBS 암호화  
로그 관리 하기  
핵심 자산인 AI 모델 학습 서버는 Pirvate Subnet에 배치하여 인터넷에서 접근 불가능하게 만들기  

- CloudWatch  
AWS상에서 사용하고 있는 시스템 서비스의 건강 상태를 실시간으로 모니터링 하게 해주는 서비스 (GPU Utillization, DB Connections, Error Rate, Inference Latency)
